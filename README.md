# IoT AI Copilot for No-Code Recovery Systems

## Overview
This is the backend for the "IoT AI Copilot" project. It provides a REST API to interpret natural language commands, generate control logic via AI, validate it for safety, and apply it to Raspberry Pi GPIO pins.

## Tech Stack
- **Language**: Python 3.9+
- **Framework**: FastAPI
- **Database**: Supabase (PostgreSQL)
- **Auth**: Supabase Auth (JWT)
- **Hardware**: Raspberry Pi (RPi.GPIO)

## Setup Instructions

### 1. Prerequisites
- Python 3.9 installed
- Raspberry Pi (optional, code mocks GPIO if not present)
- Supabase account

### 2. Environment Setup
Create a `.env` file in the `backend/` directory:
```bash
SUPABASE_URL="your-supabase-url"
SUPABASE_KEY="your-supabase-anon-key"
SUPABASE_JWT_SECRET="your-jwt-secret" # Optional for local verification
```

### 3. Installation
```bash
cd backend
pip install -r requirements.txt
```

### 4. Running the Server
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
The API is available at `http://localhost:8000/api`.
Interactive docs: `http://localhost:8000/docs`.

### 5. Database Schema (Supabase)
Run the following SQL in your Supabase SQL Editor:

```sql
-- Table: robot_config
create table robot_config (
  id bigint generated by default as identity primary key,
  sensor text not null,
  pin int not null,
  actuator text not null,
  rule text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Table: recovery_logs
create table recovery_logs (
  id bigint generated by default as identity primary key,
  status text not null,
  description text,
  timestamp timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Table: system_status
create table system_status (
  id bigint generated by default as identity primary key,
  state text not null,
  last_updated timestamp with time zone default timezone('utc'::text, now()) not null
);
```

## API Usage Example

**1. Describe Robot**
POST `/api/describe-robot`
```json
{
  "description": "Turn ON the fan connected to pin 17"
}
```

**2. Response**
```json
{
  "is_safe": true,
  "logic": {
    "sensor": "generic_sensor",
    "pin": 17,
    "action": "ON",
    "rule": "IF generic_sensor_value > threshold THEN set PIN_17 ON"
  },
  "message": "Logic validated successfully. Safe to apply."
}
```

**3. Recover (Apply Logic)**
POST `/api/recover`
```json
{
  "logic": {
    "sensor": "generic_sensor",
    "pin": 17,
    "action": "ON",
    "rule": "..."
  }
}
```
